## Следующее (по приоритету)

### 1. Карточка техники упражнения

Использует импортированную базу: у ~700 упражнений есть 1–2 фото и
пошаговые `instructions_en`. Дизайн-набросок:

- Вход: тап по названию упражнения в блоке тренировки и строка в
  exercise-library → полноэкранная карточка (как пикер: `fixed inset-0`,
  без роутинга) или отдельный роут `/exercise/[slug]`.
- Контент: фото (свайп между 1–2 кадрами), название RU/EN, мышцы
  (основная + вторичные), оборудование, механика, шаги техники.
- Инструкции пока EN-only. Перевод: кнопка «Перевести» → Mistral →
  результат кэшируется в `exercises.instructions_ru` (одно поле уже есть),
  квота новая `instruction_translate` ~20/день. Один раз перевёл — навсегда.
- Данные: в списках `instructions_*` не тянем (см. `EXERCISE_LIST_COLUMNS`),
  карточка догружает по id отдельным запросом.

### 2. Плейт-калькулятор

Тап по полю веса в подходе → шторка «что повесить на гриф»:
`(вес − гриф 20) / 2` разложить жадно по блинам 25/20/15/10/5/2.5/1.25.
Настройка грифа (20/15/гантели) в профиле. Чисто клиентская функция,
`plate-calc.ts` + тесты — полдня работы, польза каждый подход.

### 3. Суперсеты

Группировка 2+ упражнений в тренировке: общий блок, чередование подходов,
один таймер отдыха на суперсет. Требует схему (`superset_group` в
set_entries или отдельная таблица), UI-группировку в WorkoutClient и
учёт в истории/шаблонах. Самая объёмная из троицы — планировать отдельно
через brainstorming → spec → plan.

## Процесс

Фичи ведутся по циклу superpowers: brainstorming → spec
(`docs/superpowers/specs/`) → plan (`docs/superpowers/plans/`) →
исполнение inline с TDD → ветка → PR. Хук pre-push гоняет audit+lint+test+build
(build работает без `.env.local` через placeholder-переменные).
