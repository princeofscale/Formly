# GymLog — UI Redesign + i18n Design Spec
Date: 2026-05-10

## Overview

Итеративный редизайн в стиле Bold спортивных приложений + локализация RU/EN + AI-фичи. Целевая аудитория — смешанная (новички и продвинутые атлеты). Подход — постепенный: сначала дизайн-система, потом страницы по одной.

## Стек

- **i18n:** `next-intl` (App Router native)
- **Стили:** Tailwind v4 + shadcn/ui (существующий стек)
- **Шрифты:** Inter (существующий) — weight 800–900 для заголовков, `font-mono` для чисел
- **AI-фичи:** Claude API (Anthropic SDK)

## 1. Дизайн-система

### Цвета

| Роль | Значение |
|------|----------|
| Акцент (primary) | `#F59E0B` (amber-500) |
| Фон приложения | `zinc-950` |
| Карточки | `zinc-900` |
| Бордер | `zinc-800` |
| Текст основной | `zinc-50` |
| Текст вторичный | `zinc-400` |

Акцентный цвет обновляется в CSS-переменных (`--color-accent`), используется везде вместо текущего синего.

### Типографика

- **H1 страниц:** Inter 900, `text-4xl–text-6xl`, uppercase, `tracking-wider`
- **Карточные заголовки:** Inter 700, `text-xl`
- **Числа (вес, повторения, рекорды):** `font-mono`, крупно
- **Тело:** Inter 400/500, без изменений

### Компоненты

**Кнопки:**
- `rounded-sm` вместо `rounded-lg`
- Текст uppercase + `tracking-wider`
- Primary: оранжевый фон, чёрный текст
- Размер default увеличить до `h-11`

**Карточки (акцентные):**
- Левая цветная полоса `w-1` через `border-l-4 border-amber-500`
- Применяется на карточках статуса и рекордов

**Анимации:**
- Появление страниц: `fade-in` 150ms
- Hover карточек: `hover:scale-[1.01] transition-transform`
- Фокус инпутов: оранжевая обводка `ring-amber-500`

## 2. Страницы входа и регистрации

### Структура

Сплит `grid grid-cols-2`, на мобильных (`< md`) — только правая колонка.

### Левая панель (визуальная)

```

```

Содержимое:
- Логотип GymLog (иконка + название) сверху

  - RU: «Каждый подход приближает тебя к цели»

- 3 буллета фич (иконка Lucide + текст, переведены):
  - 🏋️ Логируй тренировки за секунды
  - 📈 Отслеживай личные рекорды
  - 🧠 AI-рекомендации по прогрессии

### Правая панель (форма)

```
flex flex-col justify-center px-12
```

Содержимое:
- Переключатель `RU | EN` — правый верхний угол панели
- Заголовок: Inter 900, uppercase, `text-4xl`
  - Login: «ВОЙТИ» / «SIGN IN»
  - Register: «РЕГИСТРАЦИЯ» / «CREATE ACCOUNT»
- Подзаголовок: `text-zinc-400`
  - Login RU: «Рады снова видеть тебя» / EN: «Welcome back»

- Кнопка: полная ширина, оранжевая, uppercase, `h-12`
- Ссылка переключения login↔register

### Отличие register от login

Только одно дополнительное поле — ничего лишнего (YAGNI).

## 3. Дашборд

### Шапка

```
flex items-center justify-between
```

- Левая: приветствие («Привет, {email}» / «Hey, {email}») + текущая дата
- Правая: кнопка «ТРЕНИРОВКА» / «WORKOUT» — оранжевая, крупная

**Полоса быстрых статов** (под шапкой, 3 колонки):

| Стат | Значение |
|------|----------|
| Тоннаж недели | сумма `total_volume_kg` за 7 дней |
| Тренировок | количество сессий за 7 дней |
| Лучший e1ПМ | максимальный `calculated_1rm` за всё время |

Числа крупные `font-mono text-2xl`, подписи мелкие `text-zinc-400 text-xs`.

### Блок «Сегодня»

Карточка с левой полосой:
- Тренировочный день: `border-l-4 border-green-500`, иконка Dumbbell, текст «День тренировки» / «Training Day»
- День отдыха: `border-l-4 border-zinc-600`, иконка Moon, текст «День отдыха» / «Rest Day»
- Следующая тренировка: «Следующая: {день}» / «Next: {day}»

### Лента активности

Последние 3 сессии. Каждая карточка:
- Дата `font-mono` + день недели
- Теги упражнений (из `set_entries` → `exercises.name`): `Присед · Жим · Тяга`
- Тоннаж + длительность справа
- `hover:scale-[1.01]`, клик → `/history/{id}`

Получение тегов: запрос distinct `exercise_id` для сессии → join `exercises.name` → первые 3.

### Мышечный хитмап

Существующий компонент `MuscleHeatmap`. Добавить справа легенду топ-3 мышц:
```
Грудь     12 подходов  ████████
Спина      8 подходов  ██████
Квадрицепс 6 подходов  ████
```

### Быстрые рекорды

Горизонтальный скролл (`overflow-x-auto flex gap-3`). Карточка рекорда:
- Название упражнения
- e1ПМ крупно `font-mono`
- Стрелка ↑ если рекорд побит за последние 30 дней

Данные: топ-5 упражнений по частоте использования + их максимальный `calculated_1rm`.

## 4. Профиль

### Шапка

- Аватар: круг `w-20 h-20`, инициалы из email на `bg-amber-500`
- Email крупно
- Кнопка «РЕДАКТИРОВАТЬ» / «EDIT» — ghost, переключает inline-режим

### Стат-полоса (3 колонки)

| Стат | Источник |
|------|----------|
| ИМТ | `calculateBMI` |
| Стаж | `training_since` → годы |
| Локация | `training_location` переведённая |

### Секция «Физические данные»

Сетка 2×2. Каждый параметр:
```
<число> <единица>
<подпись>
```
В режиме редактирования → Input с `autoFocus`.

### Секция «Расписание»

7 кнопок-таблеток горизонтально:
- Активный: `bg-amber-500 text-black`
- Неактивный: `bg-zinc-800 text-zinc-400`
- Подпись: «Следующая тренировка: {день}» / «Next training: {day}»

### Секция «Язык»

Два таба `RU | EN`:
- Активный: `bg-amber-500 text-black rounded-sm px-4 py-1`
- Неактивный: `bg-zinc-800 text-zinc-400`

Сохранение: `profiles.locale` в Supabase + cookie `locale` для pre-auth.

### Секция «Опасная зона»

Разделитель `border-t border-zinc-800 mt-8`. Две кнопки:
- «Выйти» / «Sign out» — ghost
- «Удалить аккаунт» / «Delete account» — destructive, confirm dialog

## 5. Локализация (next-intl)

### Установка

```bash
npm install next-intl
```

### Структура файлов

```
messages/
  ru.json
  en.json
src/i18n/
  routing.ts    # locales: ['ru', 'en'], defaultLocale: 'ru'
  request.ts    # cookie → Accept-Language → 'ru'
```

### Алгоритм определения локали

1. Cookie `locale` (установлен пользователем)
2. `profiles.locale` из Supabase (если залогинен)
3. `Accept-Language` заголовок браузера
4. Дефолт: `'ru'`

### Структура без изменения URL

Используем **без prefix в URL** (`/dashboard` не `/ru/dashboard`) через `next-intl` middleware с cookie-based локалью. Это проще и не ломает существующие ссылки.

### Схема messages/ru.json

```json
{
  "auth": {
    "login": { "title": "ВОЙТИ", "subtitle": "Рады снова видеть тебя", ... },
    "register": { "title": "РЕГИСТРАЦИЯ", "subtitle": "Начни отслеживать прогресс", ... },
    "slogan": "Каждый подход приближает тебя к цели",
    "features": ["Логируй тренировки за секунды", "Отслеживай личные рекорды", "AI-рекомендации по прогрессии"]
  },
  "nav": { "home": "Главная", "history": "История", ... },

  "days": { "mon": "ПН", "tue": "ВТ", ... }
}
```

### Новое поле в Supabase

```sql
ALTER TABLE profiles ADD COLUMN locale text DEFAULT 'ru';
```

## 6. Новые AI-фичи (приоритет по сложности)

### Фича 1: Умная прогрессия (быстро)

Расширение существующего `progression.service.ts`:
- Анализ RPE за последние 4 недели по упражнению
- Если RPE растёт (>8.5 в среднем) без роста веса → рекомендация deload
- Если все подходы в топе диапазона → рекомендация +2.5/5кг (уже есть)
- Отображение после завершения тренировки

### Фича 2: AI-план тренировок (средняя сложность)

- Новая страница `/plan` или модал после онбординга
- Пользователь выбирает цель: «Сила» / «Объём» / «Похудение»
- Запрос к Claude API с контекстом: стаж, расписание, история сессий (топ-5 упражнений + прогресс)
- Claude возвращает недельный план: упражнения + подходы + повторения + RPE
- План сохраняется в `workout_plans` таблице

### Фича 3: Предсказание усталости (сложно)

- Еженедельный расчёт «Индекса усталости» на основе:
  - Тоннаж vs Volume Landmarks (MV/MEV/MRV)
  - Динамика RPE за 2 недели
  - Частота тренировок vs расписание
- Отображается на дашборде как индикатор `Свежий / Устал / Перебор`
- При `Перебор` → рекомендация взять 2-3 дня отдыха
