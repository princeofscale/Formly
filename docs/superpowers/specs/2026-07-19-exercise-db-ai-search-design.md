# Большая база упражнений + ИИ-поиск «Возможно, вы имели в виду…»

Дата: 2026-07-19
Статус: одобрено

## Проблема

В зале пользователь вбивает свой тренажёр — его часто нет в базе (~84 упражнения).
Приходится каждый раз создавать кастомное упражнение или бросать затею. Нужны:

1. Большая база упражнений и тренажёров с нормальными русскими названиями и
   сленговыми алиасами («хаммер», «гравитрон», «гак»).
2. ИИ-подсказка в поиске: на непонятный запрос — «Возможно, вы имели в виду…»
   с реальными упражнениями из базы.

## Решение (одобренный вариант: «Гибрид»)

### 1. Импорт free-exercise-db

- Источник: <https://github.com/yuhonas/free-exercise-db> — ~873 упражнения,
  public domain (Unlicense), у каждого 1–2 фото.
- Одноразовый скрипт `scripts/import-free-exercise-db.mjs`:
  скачивает `dist/exercises.json`, маппит поля на наши енумы
  (`primaryMuscles`/`secondaryMuscles` → `muscle_group`, `equipment` →
  `equipment`, `mechanic` → `mechanic`), генерирует SQL-миграцию
  `supabase/migrations/*_import_free_exercise_db.sql`.
- Значения енумов, которых у нас нет, добавляются той же миграцией
  (прецедент: `20260528120000_exercises_enum_extend.sql`).
- **Русские названия** (`name_ru`) генерируются для всех импортируемых
  упражнений (перевод выполняет Claude при генерации миграции, не runtime).
- **Кураторские алиасы** для топ-300 ходовых упражнений: сленг и разговорные
  названия, нижний регистр («хаммер», «гравитрон», «гак-присед», «кроссовер»,
  «бабочка», «молотки»…). Формат — существующая колонка `aliases text[]`.
- **Дедупликация**: нормализованное имя (lowercase, trim) сравнивается с
  существующими 84; дубликаты не вставляются, но их фото копируются в
  `image_urls` существующей строки, если она пустая.
- Фото — прямые URL на `raw.githubusercontent.com/yuhonas/free-exercise-db/…`
  (поиск уже рендерит превью из `image_urls[0]`).
- Импортированные строки: `is_custom = false`, `created_by = null`,
  `instructions_en` — из источника, `instructions_ru` — не переводим (объём;
  вне scope).

### 2. Fuzzy-поиск (pg_trgm, без ИИ)

- Миграция: `create extension if not exists pg_trgm` + GIN-индексы
  `gin_trgm_ops` на `name` и `name_ru`.
- RPC-функция `search_exercises_fuzzy(q text)` (security invoker, RLS
  действует): top-5 по `greatest(similarity(name, q), similarity(name_ru, q))`
  с порогом `> 0.3`, только `is_custom = false or created_by = auth.uid()`.
- `searchExercises` в `src/lib/db/exercises.ts`: если ILIKE/alias-поиск дал
  пусто — второй запрос через RPC. «жым лижа» → «Жим лёжа» мгновенно, без квот.

### 3. ИИ-подсказка (Mistral)

- Новый сервис `src/lib/services/exercise-suggest.service.ts` по образцу
  `exercise-alternatives.service.ts` (одобренный вариант: **выбор из
  пронумерованного списка**):
  - Вход: запрос пользователя + весь каталог компактными нумерованными
    строками `№|name|name_ru|мышца|оборудование` (~950 строк ≈ 10k токенов).
  - Модель: `mistral-large-latest`, `responseFormat: json_object`, низкая
    температура.
  - Выход: `{"items":[{"index":123,"reason":"короткая причина, ≤12 слов"}]}`,
    максимум 3.
  - Сервер маппит номера в упражнения; невалидные/повторные номера
    отфильтровываются — галлюцинации невозможны по построению.
  - Чистые функции (тестируемые без сети): `serializeCatalog(exercises)`,
    `parseSuggestions(raw, catalogLength)`.
- Серверный экшен `suggestExercisesAction(query)` в
  `src/app/(app)/workout/[id]/actions.ts`:
  `verifySession` → `consumeAiQuota(supabase, userId, 'exercise_suggest')`
  (новый вид, лимит 30/день) → каталог (компактные колонки) → сервис →
  `{ items: { exercise, reason }[] }`.
- Ошибки не пробрасываются клиенту как исключения: квота исчерпана, Mistral
  недоступен, невалидный JSON → `{ items: [] }` (плюс лог на сервере).

### 4. UI: `src/components/workout/ExerciseSearch.tsx`

Поток:

1. ≥2 символов → `searchExercisesAction` (теперь внутри с fuzzy-фолбэком).
2. Выдача пуста **и** `navigator.onLine` → таймер 800 мс после последнего
   ввода → один вызов `suggestExercisesAction` на уникальный запрос
   (повторный ввод той же строки не дергает ИИ повторно).
3. Пока ждём: строка «ИИ подбирает варианты…» со спиннером.
4. Ответ непуст: заголовок «Возможно, вы имели в виду…», 2–3 карточки
   (превью, название по локали, причина от ИИ, мышца). Тап — упражнение
   выбрано, как обычный результат поиска.
5. ИИ вернул пусто / ошибка / офлайн: существующий CTA «создать своё»
   (ExerciseForm) — без изменений.

i18n (`messages/ru.json`, `messages/en.json`): `aiSearching`
(«ИИ подбирает варианты…»), `aiDidYouMean` («Возможно, вы имели в виду…»).

### 5. Производительность

База вырастает до ~950 строк, `instructions_en` у импортированных — длинные.
Горячие выборки перестают использовать `select('*')`:

- `getExercises` (страница тренировки `allExercises`, workout/new,
  exercise-library) — только колонки списка: `id, name, name_ru, slug,
  primary_muscle, secondary_muscles, mechanic, equipment, is_custom,
  created_by, aliases, image_urls`.
- `searchExercises` — те же колонки.

Иначе страница тренировки тяжелеет на ~1 МБ.

## Обработка ошибок

| Ситуация                     | Поведение                                     |
| ---------------------------- | --------------------------------------------- |
| Квота `exercise_suggest` 30/день исчерпана | ИИ-блок не показывается, CTA «создать своё» |
| Mistral упал / таймаут / кривой JSON | то же, ошибка в серверный лог           |
| Офлайн                       | ИИ не вызывается вовсе                        |
| ИИ вернул левые номера       | отфильтрованы `parseSuggestions`              |
| Запрос из спецсимволов       | существующий `sanitizeFilterTerm`             |

## Тестирование

- Юнит (vitest, без сети): `serializeCatalog` (формат строк, экранирование
  `|`), `parseSuggestions` (валидные индексы, отброс мусора, кап 3, повторные
  индексы), логика маппинга/дедупа импорт-скрипта (нормализация имён,
  маппинг енумов) — скрипт экспортирует чистые функции.
- Существующие тесты остаются зелёными.
- Ручная проверка: «жым лижа» → fuzzy; «жим в хаммере на грудь» → ИИ-карточки;
  тап по карточке добавляет упражнение; офлайн → сразу CTA.

## Вне scope

- Перевод `instructions_ru` для импортированных упражнений.
- Страницы деталей упражнений, видео, заметки.
- Редизайн exercise-library.
