# Полноэкранный мобильный пикер упражнений

Дата: 2026-07-20
Статус: одобрено
База: ветка `feat/exercise-db-ai-search` (нужны алиасы и `suggestExercisesAction` из PR #14)

## Проблема

Поиск упражнений на странице тренировки — узкий инлайн-дропдаун под липкой
шапкой. На телефоне клавиатура перекрывает список, строки мелкие, «частые
упражнения» видны только в пустой сессии. Продуктом пользуются в зале с
телефона.

## Решение

### Вход

В липкой шапке вместо `<ExerciseSearch>` — крупная кнопка «+ Упражнение»
(`tar-cta`, во всю ширину). Тап открывает полноэкранный оверлей
(`fixed inset-0 z-50`), скролл body блокируется.

### Пикер (сверху вниз)

1. Шапка (закреплена): кнопка ✕ + поле поиска с автофокусом.
2. Чипы групп мышц (горизонтальный скролл): Все · Грудь · Спина · Плечи ·
   Бицепс · Трицепс · Ноги · Пресс · Прочее. Укрупнённые маппинги:
   Спина = back+lats+traps; Плечи = 3 дельты; Ноги = quads+hamstrings+glutes+calves;
   Прочее = forearms+cardio.
3. Контент (скроллится):
   - Запрос < 2 символов: секция «Недавние» (проп `recentExercises` — топ-6
     за 30 дней, уже считается сервером; фильтруется чипом и `sessionExerciseIds`),
     затем «Все упражнения» по алфавиту локали.
   - Запрос ≥ 2 символов: мгновенная локальная фильтрация по `allExercises`
     (они уже на клиенте): name + name_ru + aliases, регистронезависимо,
     ё→е; совпадения с начала строки ранжируются выше подстрочных; лимит 50.
     Ноль сетевых запросов — работает офлайн.
   - Локально пусто: ИИ-блок «Возможно, вы имели в виду…»
     (существующий `suggestExercisesAction`, дебаунс 800 мс, только онлайн,
     один вызов на запрос) + CTA «создать своё» (существующая `ExerciseForm`).
4. Строка: фото 40px, название по локали, мышца; высота ≥ 56px (под палец).
   Уже добавленные в сессию — галочка, приглушены, недоступны.
5. Выбор упражнения закрывает пикер. Закрытие: ✕, Escape.

### Производительность

Список ~800 строк рендерится без виртуализации: `content-visibility: auto` +
`contain-intrinsic-size` на строках — браузер не раскладывает то, что за
экраном.

### Код

- Новое: `src/components/workout/ExercisePicker.tsx`,
  `src/lib/utils/exercise-filter.ts` (+ тесты) — чистые
  `filterExercises(all, query, chip, limit)`, `matchesChip`, `normalizeQuery`,
  `MUSCLE_CHIPS`, `CHIP_MUSCLES`.
- Изменяется: `WorkoutClient.tsx` (кнопка вместо инлайн-поиска; в пикер
  уходят `allExercises`, `suggestedExercises`, id упражнений сессии),
  `messages/ru.json`, `messages/en.json` (ключи `workout.*`: addExercise,
  recent, allExercisesSection, added, chip*).
- Удаляется: `src/components/workout/ExerciseSearch.tsx` (ИИ-блок и
  создание кастомного переезжают в пикер).

## Тестирование

Юнит (vitest): нормализация ё/регистр, поиск по алиасу («гакк»),
приоритет startsWith над contains, фильтр чипом, лимит, пустой запрос → [].
Ручное: открыть пикер на телефоне, чипы, «Недавние», поиск «жим», выбор,
галочки добавленных, ИИ-фолбэк на «жим в хаммере на грудь стоя», офлайн.

## Вне scope

Виртуализация списка, карточка техники упражнения, drag-переупорядочивание.
